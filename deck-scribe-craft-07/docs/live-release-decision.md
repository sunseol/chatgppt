Path: `docs/live-release-decision.md`

Release decision: Blocked

## Decision basis

The current worktree has stronger Live-readiness contracts than the previous mock-provider build: production mock selection is blocked by policy, provider provenance and structured Codex output gates exist, research fetches preserve capture metadata, image provider responses preserve request metadata, live generation report lineage is checked per slide, Golden Path E2E and manual QA evidence bundles have local verifiers, DF-247 final release gate criteria are encoded, and the final export gate rejects contaminated production lineage. It still does not yet have enough Live evidence to approve the Live Initial Version.

## Blocking facts

- P0 Live tickets DF-210 through DF-247 are not all Verified Live; DF-241 still lacks a signed `live_e2e_report.md`, screenshots/recording, and final validation bundle from a real run.
- Production package mock execution paths are blocked at the policy layer and the current dry-run package scan no longer finds mock provider or mock stage resources.
- Live benchmark pass count is 0 of 5, below the required 4 of 5.
- Golden Path lineage with zero mock or fixture artifacts, text turn ids, image request ids, prompt versions, and compositor-matched export hashes is enforced by local gates but has not been produced by a real run.
- Clean-machine Codex login, image credential setup, live interview, live research, live image generation, title edit, and export have not been recorded.
- The current DMG is unsigned and not notarized.
