# design_system v1

## Purpose
Create a consistent design system for the approved deck plan.

## Inputs
- brief artifact id
- deck plan artifact id
- aspect ratio
- language

## Output Format
- canvas
- grid
- colors
- typography
- layout rules
- component rules
- visual language
- negative rules

## Rules
- Use a restricted token set.
- Keep text legible at slide scale.
- Avoid one-note palettes.
- Do not introduce unsupported fonts.

## Failure Mode
Return blocked status when the plan or brief is not approved.
